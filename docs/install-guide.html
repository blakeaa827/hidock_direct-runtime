Before you start

HiDock Direct is made of four pieces working together. Your HiDock holds the recordings, the app pulls them off and hands them to AssemblyAI, AssemblyAI returns the transcript, and everything lands in an archive folder on your Mac. Claude Code does the install and setup for you.

Device
HiDock
Records your calls over USB.
→
App
HiDock Direct
Pulls recordings off the device.
→
Service
AssemblyAI
Transcribes and labels speakers.
→
Output
Archive
Transcripts on your Mac.

What you need

  • A Mac you can install software on. You will need your computer password once, during Homebrew install.
  • Your HiDock and its USB cable. Connected to the Mac when you run the app.
  • A personal email address. For GitHub, so the account stays yours regardless of employer.
  • A paid Claude plan. Claude Code needs Claude Pro (about 20 dollars a month).

One connection to the device at a time

The HiNotes.ai web app (the factory software for the device) and HiDock Direct both talk to the device over the same USB connection, and only one can hold it at a time. Before you run HiDock Direct, close the HiNotes.ai browser tab, or log out, so it releases the device. If HiNotes.ai still holds the connection, HiDock Direct cannot read the device.

Walkthrough

Install, step by step

01

Create a GitHub account

GitHub hosts the HiDock Direct code. You need a free account to get access.

  • Go to github.com and start the sign-up.
  • Sign in with Google, or use your personal email. Do not use your work email.
  • Pick a username and write it down. You can change it later if you want.

Skip the Copilot upsell

GitHub will offer Copilot during sign-up. You do not need it for this. Decline it.

02

Get access to the repository

HiDock Direct lives in a private repository. The owner adds your GitHub username as a collaborator, then you accept the invite.

  • Send the repository owner your exact GitHub username.
  • Once they add you, open your GitHub inbox (the icon at the top right, look for the blue dot next to your profile).
  • Open the invitation and click Accept invitation.
03

Install Homebrew, Git, and Python

Homebrew is a package manager for macOS. You will use it to install Git and Python. Claude Code needs Git to work with local repositories, and HiDock Direct needs Python 3.11 or newer, so set both up before you install Claude Code.

Open Terminal. Press ⌘ Space, type Terminal, and press Return.

Install Homebrew. Paste this into Terminal and press Return, then enter your Mac password when asked. This is the official one-line installer from brew.sh:

Terminal
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

The script it fetches is always the current one, so this command does not go stale.

The password looks like nothing is happening

When you type your Mac password, the characters stay hidden and the cursor does not move. That is normal. Type it and press Return. Then wait for the prompt to come back before continuing.

Add Homebrew to your PATH. When the install finishes, Homebrew prints a short Next steps block with two or three commands. Run each of those lines in Terminal, one at a time. They look like this, with your own username in the path:

Terminal
echo >> /Users/yourname/.zprofile
echo 'eval "$(/opt/homebrew/bin/brew shellenv)"' >> /Users/yourname/.zprofile
eval "$(/opt/homebrew/bin/brew shellenv)"

Use the exact lines Homebrew printed for you. The path and username are specific to your Mac.

Confirm it worked. Open a fresh Terminal window (⌘ N) and run:

Terminal
brew --version

Install Git. Run this, then type y to proceed:

Terminal
brew install git

Install Python. HiDock Direct needs Python 3.11 or newer. Install a current version with Homebrew:

Terminal
brew install python@3.12
04

Install Claude Code

Claude Code is the assistant that runs the rest of this install for you. It reads the repository's instructions, installs what it needs, and configures your machine.

  • Go to Anthropic's Claude Code page (claude.com/claude-code) and click Get started, then Download for Mac.
  • Open the downloaded .dmg file, then drag Claude into your Applications folder.
  • Open Claude. When macOS warns that it was downloaded from the internet, click Open.
  • Sign in. Pick any role on the setup screen to get past it, then open the Code section.

Claude Code needs a paid plan

It will not run on the free tier. Claude Pro is about 20 dollars a month, or roughly 17 a month if you pay annually.

Switch the model to Haiku

In the bottom right of the Code view, click the model name (it says Sonnet) and choose Haiku. Haiku uses far fewer credits, which matters on the 20 dollar plan. If the install hits a snag it cannot get past, switch back to Sonnet for that part, then return to Haiku for everyday use.

05

Turn on Claude Code's permissions

For Claude Code to do the rest of the setup on its own, it needs permission to act on your computer. Set this up now, right after installing Claude, so every step that follows runs without stopping to approve each command. Quit and reopen Claude, then turn these on so the grants take effect.

Open Claude's settings (click your name at the bottom left, then Settings). The important switches:

  • Under Desktop app › General: turn on Allow browser actions and Enable computer use. Grant Accessibility and Screen Recording when prompted. Each opens System Settings, where you switch Claude on.
  • Under Capabilities: turn on the options you want, including Generate memory from chat history, Search and reference chats, Load tools when needed, and Switch models.
  • Under Claude Code: turn on Allow bypass permissions mode.

Restart Claude again so Screen Recording takes effect.

Stop approving every action by hand

In the message bar, set the mode to Accept edits, then Bypass permissions. Now Claude can finish the install without stopping to ask each time. The exact menu names can shift between versions, so look for the closest match.

06

Create an AssemblyAI account

AssemblyAI does the transcription. New accounts come with free credits, which is plenty to get everything working before you ever need to add more.

  • Go to assemblyai.com and click Get started, then sign up.
  • On the dashboard, find your API key at the top.

Leave this tab open

You will copy this API key into Claude Code in Step 9. Do not click around or regenerate it.

07

Set up GitHub SSH access

SSH is how your Mac proves to GitHub that it is you, without typing a password every time. Claude Code sets this up for you.

In the Claude Code message box, paste this and press Return:

Claude Code prompt
I just set up a new GitHub account. Please set up my local computer for GitHub SSH access for the CLI.
  • When Claude asks for a workspace folder, create a new folder named claude and trust it.
  • With permissions already on from Step 5, Claude runs these actions on its own without stopping to ask.

Git is already in place

Because you installed Git in Step 3, Claude Code can work with your local repository without stopping to ask for it.

08

Add your SSH key to GitHub

Now connect the key Claude generated to your GitHub account. Reopen your SSH conversation in Claude (type continue if it paused) and find the key it created.

  • In Claude Code, find the public key (the line that starts with ssh-) and click its copy button.
  • In GitHub, go to Settings › SSH and GPG keys › New SSH key.
  • Give it a name (like My Mac), paste the key, and click Add SSH key.

Back in Claude Code, test the connection:

Claude Code prompt
SSH key has been added. Please test it.

Once it confirms success, your Mac can talk to GitHub.

09

Install the HiDock Direct app

Now Claude installs the app for you, following the instructions in the repository and placing it in a permanent folder in your home directory.

  • Open the HiDock Direct repository on GitHub and copy its URL from the address bar.
  • Paste this prompt into Claude Code, with the repository URL at the end:
Claude Code prompt
Install the app and follow the instructions in the README at this link: [paste the repository URL]. Install it into a folder named hidock-direct in my home directory (~/hidock-direct), not a temporary folder. Then add an alias to my shell profile so I can launch it by typing hidock in any Terminal.

The ~ in ~/hidock-direct is the tilde (Shift plus the key just left of 1). It is shorthand for your home folder.

  • When Claude asks for your AssemblyAI API key, switch to the AssemblyAI tab, copy the key, and paste it in.
  • Claude also sets up a short hidock command, so you can start the app by typing one word.
  • Watch progress by clicking the running task indicator. When it finishes, Claude shows you how to run the app.

What Claude sets up here

libusb (for talking to the device over USB), a Python virtual environment (using the Python you installed in Step 3), and your AssemblyAI key, all wired together so the app can run.

10

Clear any running instances

Only one copy of HiDock Direct should run at a time. If one is already running, stop it first.

  • Open a Terminal and type hidock. If it says another instance is running, press ⌘ Q to quit that window.
  • Then ask Claude to clean up anything left over:
Claude Code prompt
Kill all instances of HiDock running on my computer.
11

Run HiDock Direct and verify

Start the app and let it work. Make sure your HiDock is connected and the HiNotes.ai web app is closed (see the warning at the top).

Open a Terminal and run:

Terminal
hidock

The app connects to your HiDock, pulls the recordings off it, and sends each one to AssemblyAI to transcribe. It keeps running and watches for new recordings, so leave the window open while you work.

Find your transcripts:

  • HiDock Direct saves transcripts to their own archive folder in your home directory, separate from the app folder. Open Finder and go to your home folder (⌘ Shift H), then open the hidock-archive folder.
  • Drag the archive folder into your Finder favorites so it is always one click away.
  • Switch the view to List and click the Name column header to sort; click it again so the newest dates sit on top (filenames start with the date). Transcripts are grouped into folders by month.
Reference

Understanding your transcripts

Each recording produces three files in the archive. Only the first one matters for daily use.

FileWhat it is
.md
The transcript. Speaker-labeled text (Speaker 1, Speaker 2, and so on). Open it with TextEdit to read it, or paste it into any AI tool. This is the file you will actually use.
.mp3
The original audio recording, in case you want to listen back.
.aai.json
The raw transcript data AssemblyAI returned. Its presence also tells HiDock Direct the recording is already done, so it does not transcribe it twice. You can ignore it.
Tip

Putting names to speakers

The transcript labels speakers as Speaker 1, Speaker 2, and so on, not by name. If you want real names, paste the transcript into an AI tool along with the list of people who were on the call (the attendee list from the meeting invite works well) and ask it to match each speaker to a name based on context. It picks up on moments where someone is addressed by name and assigns the rest from there.

You usually do not need names

For pulling out topics, requirements, and what was agreed, what matters is what was said, not who said it. Naming speakers is optional.

Day to day

Your daily workflow

Each morning, connect your HiDock, make sure the HiNotes.ai web app is closed, open a Terminal, and type hidock. Leave it running. As you take calls through the day, HiDock Direct notices each new recording, pulls it off the device, and transcribes it automatically. Your transcripts collect in the archive folder, ready whenever you need them.
